关于 'Mirror' 的一些说明

LoeiFy Updated at Jan 15, 2016

我个人是不喜欢写文章相关的，我感觉自己文字组织有问题。虽然有打理 http://guo.lu/ 这个几乎就是文章的网站，但里面的文章都是另一个作者发的。网站初期还有发一些设计相关的收集，但后面纯粹就是另一个作者发文章了，我只是复杂网站设计开发相关了。

虽然不怎么写文字，但是我还是有时候想记录一些事情或者记录一些代码相关。所以我想选择一些轻量博客程序用来记录文字，例如 hexo，但是我还是觉得需要一个本地环境麻烦。虽然现在已经有很多在线写博客相关，但是高级功能需要付费或者自定义不是那么好。

直到看到 v2ex 上的一篇讨论 https://www.v2ex.com/t/198021 我发现这个才是我想要的东西： 我只需要在 issue 上写文章，通过 GitHub API 把我的文章自定义显示出来，我只另外需要的是一个域名。

Blog 地址

http://mirror.am0200.com/

用到的 GitHub API

  • 获取用户相关信息
GET /users/:username
  • 获取某个 repo 的 issues 
GET /repos/:owner/:repo/issues
  • 获取某个 repo 的单个 issue 
GET /repos/:owner/:repo/issues/:number
  • 获取某个 issue 的 comments
GET /repos/:owner/:repo/issues/:number/comments

相关插件

Zepto https://github.com/madrobby/zepto

Marked https://github.com/chjj/marked

highlight.js https://github.com/isagalaev/highlight.js

扩展

  • GitHub API 同时也提供了搜索功能，所以后期可以集成搜索功能，以及分类功能
  • 现在列评论可以直接通过 API 获取，但是添加评论还是需要到 GitHub 上评论，虽然有 API 可以直接添加评论，但这涉及登录认证问题，这个只能忽略
  • 由于页面是由 JS 生成，SEO 功能全无

名字来源

Mirror 是 DJ Okawari 所作的专辑
